# Contributing to The Dependency Installer

So you want to `npm install contribution`? Great news: this package has zero vulnerabilities. (Unlike `self-confidence`, which has 47.)

## Getting Started

```bash
git clone https://github.com/arnoldwender/the-dependency-installer.git
cd the-dependency-installer
npm install
npm run dev
```

The app should be running at `http://localhost:5173`. Start installing life packages.

## How to Contribute

1. **Fork the repo** and create a feature branch (`feat/new-life-package`)
2. **Make your changes** — new packages, funnier install logs, better dependency trees
3. **Test locally** — make sure your life package actually installs (or hilariously fails)
4. **Submit a PR** describing your changes

## Code Style

- TypeScript everywhere — type safety is the one dependency that always resolves
- React functional components with hooks
- Tailwind CSS for styling
- Life package humor should be relatable and universal
- Install logs should feel authentic — anyone who's stared at npm output should feel at home

## Ideas We Love

- New life packages (`financial-literacy`, `parallel-parking`, `small-talk@social`)
- Creative install failures and error messages
- Dependency conflicts between life packages ("motivation conflicts with netflix@latest")
- Funny vulnerability scan results
- New npx life commands

## Bug Reports

Open an issue. Describe what happened. If `work-life-balance` somehow installed successfully, that's definitely a bug.

## Feature Requests

Got an idea for a life package the world needs? Open an issue with the `enhancement` label. Describe the package name, what it does, and what vulnerabilities it might have.

## The Golden Rule

The humor here is about the shared absurdity of modern life, told through the lens of something every developer knows: npm. Keep it lighthearted and relatable.
